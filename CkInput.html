RDF :

Définition :

Resource Description Framework est un modèle de graphe étiqueté et orienté qui sert à décrire les ressources web et leurs métadonnées, ce modèle se base sur une forme de triplet. Les triplets contiennent un sujet, objet et un prédicat, sachant que le prédicat est une propriété, alors que les deux autres sont des ressources. RDF est devenu une recommandation de W3C depuis le 10 février 2004. [W3C, 2013]

Triplet :

D’après le W3C, l’explication des triplets est la suivante :

  • Ressource : est tout ce qui peut avoir une URI, comme « http://www.w3schools.com/rdf »
  • Propriété : est une ressource qui a un nom, comme « auteur » ou « page d’accueil ».
  • Valeur : est la valeur d’une propriété, tels que « Jean Edouard » ou « http://www.w3schools.com », on note que la valeur de la propriété peut être une autre ressource.

Langage d’ontologie :

Définition :

Le langage de spécification est l’élément central sur lequel repose l’ontologie.

La plupart de ces langages se basent sur ou sont proches de la logique du premier ordre, et représentent donc les connaissances sous forme d’assertion (sujet, prédicat, objet). Ces langages sont typiquement conçus pour s’abstraire des structures de données et se concentrer sur la sémantique.

Les Langages pour l’ontologie:

RDFS :

RDFS est un langage extensible de représentation des connaissances, il fournit des éléments de bases pour la définition d’ontologies ou vocabulaires destinés à structurer des ressources RDF. En plus, Le Schéma RDF (RDFS) définit un vocabulaire de ressources RDF qui peut être utilisé pour décrire des propriétés sur d’autres ressources RDF.
